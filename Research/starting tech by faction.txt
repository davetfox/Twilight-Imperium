# Twilight Imperium Starting Technologies by Faction

## What this appendix adds

This appendix adds the missing **faction-by-faction starting technology table** to the earlier card-and-phase research. It covers the current official product line for **Twilight Imperium Fourth Edition** as it exists across the **base game**, **Prophecy of Kings**, **Codex content**, and **Thunder’s Edge**. Fantasy Flight’s current product page lists the four Codex volumes, the separate **Alternate Technology Cards** file, and the **Codex 4.5 Technology Card Updates** file as official support resources, while the Thunder’s Edge announcement states that the expansion adds **five new factions**, gives **all factions** new reference cards that show their starting units and technology, and compiles previously print-and-play Codex material into the official line. citeturn11view0turn12view0

For the faction-by-faction entries themselves, the best publicly indexed source is the current **Starting Technology** reference page on the Twilight Imperium Wiki. That page explicitly says the information “can be found on the back of the faction sheet,” and that players place those technology cards during **“Gather Starting Components.”** I use that page as the line-by-line faction source, and I cross-reference the timing framework against the official **Rules Reference**. citeturn2view0turn3view0turn16view0

## Rules framework for setup and phase mapping

The official setup rule is straightforward: in **Step 11 — Gather Starting Components**, each player gains the **starting technologies** listed on the back of their faction sheet and places those cards faceup near that sheet. That means starting technologies are fundamentally **setup metadata** that then continue to matter for the rest of the game. citeturn3view0turn16view0

The official technology rules also matter for how to store and interpret this data. The Rules Reference says that a non-upgrade technology has one of four colors—**Biotic, Warfare, Propulsion, or Cybernetic**—and that those colors have no inherent effect by themselves, but they **do satisfy prerequisites** when researching future technologies. It also says **unit upgrade technologies do not have a color and do not satisfy prerequisites**. The main way players research technology is through the **Technology strategy card**, whose primary and secondary abilities resolve during the **Action Phase**. citeturn5view0

For phase mapping, I use the following rule anchors. Technologies that say “during the status phase” map to the **Status Phase**. Technologies that begin with **ACTION:** map to a **component action in the Action Phase**. Technologies that modify movement, bombardment, production, space cannon, or ground combat map to the exact tactical-action substeps described in the Rules Reference, such as **Movement**, **Space Cannon Offense**, **Bombardment**, **Ground Combat**, and **Production**. citeturn3view0

A practical Codex note belongs here. The official resource page confirms that Fantasy Flight provides both **Alternate Technology Cards** and **Codex 4.5 Technology Card Updates** as separate resources. In the current public technology index, **Magen Defense Grid** is the only fixed starting basic technology that is visibly marked with live replacement versions. I therefore treat **Arborec’s starting Magen Defense Grid as using the newest update**, while the other fixed starting basic technologies remain on their current non-Ω text unless a later official update names them specifically. That is an inference from the current public tech index and official resource list, and I am flagging it as such. citeturn11view0turn9view0

## Codex and Thunder’s Edge version notes

The official support page shows that Fantasy Flight treats technology updates as a live part of the game line, with the Codex volumes plus separate **Alternate Technology Cards** and **Codex 4.5 Technology Card Updates** resources. citeturn11view0

In the current public technology index, **Magen Defense Grid** is the only fixed starting technology that is explicitly shown with multiple replacement generations: original base-game text, **Magen Defense Grid Ω**, and **Magen Defense Grid ΩΩ**. That makes **Arborec** the clearest faction whose fixed starting technology should be flagged as having a Codex-era deprecation chain. citeturn9view0

By contrast, the current public technology index does **not** attach Ω markers to the other fixed starting basic technologies such as **Antimass Deflectors**, **Neural Motivator**, **Sarween Tools**, **Plasma Scoring**, **Gravity Drive**, **Graviton Laser System**, **Dark Energy Tap**, **Sling Relay**, **Predictive Intelligence**, **Psychoarchaeology**, **AI Development Algorithm**, or **Self-Assembly Routines**. I therefore treat those as **current/original** in this appendix unless a later official update names them specifically. Again, that is an inference from the current published index, not a quote from an official errata line. citeturn9view0turn11view0

Thunder’s Edge affects starting-tech interpretation in a different way. It does not mainly deprecate old starting technologies; instead, it adds **five more factions**, gives every faction a **breakthrough** with a two-color synergy for technology requirements, and ships **reference cards** that explicitly summarize starting technology. In database terms, Thunder’s Edge should therefore be modeled as both **new faction-starting-tech data** and a **new modifier on how tech colors can matter later in the game**. citeturn12view0

## Integration notes for the site database

If you want this appended cleanly to the larger card-and-phase dataset, the most robust model is to store starting technologies as **setup data that also points forward into later phases**. The rules basis for that approach is Step 11 of setup, where players gain those technologies before round 1 begins. citeturn3view0turn16view0

A build-friendly structure would look like this conceptually:

```text
faction
sourceSet
startingTechMode
startingTechs
choicePool
hasNoStartingTech
setupTiming
currentVersionNotes
laterPhaseImpacts
sourceCitation
```

The fields that matter most for correctness are **startingTechMode** and **currentVersionNotes**. Those let you distinguish a fixed start like **Sol = Neural Motivator + Antimass Deflectors**, a null start like **Sardakk N’orr = none**, a bounded choice like **Argent Flight = choose 2 of 3**, and a fully dynamic start like **Council Keleres** or **Deepwrought Scholarate**. citeturn2view0turn12view0
